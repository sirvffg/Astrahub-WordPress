# 更新日志

## 0.2.0 - 2026-07-12

- 重构星际通讯面板界面，与 Halo 端界面逻辑和视觉风格完全对齐：新增星链成员侧栏（搜索/分页）、首次使用同意门（滚动至底部解锁）、提及/引用快捷菜单、成员资料弹窗（含 RSS 最近文章与助手能力面板）、表情包管理（上传/删除/分类切换）。
- 重构友链列表请求：后端统一使用 `/v1/friend-invitations/overview` 端点，新增 `tab`/`limit`/`offset` 参数校验；前端对齐参数行为，始终携带钳制后的分页参数。
- 统一世界频道 API 请求模式，使用 `request<T>` 通用签名代理（`/hub/get`、`/hub/post`），对齐 Halo 端 `WorldChatService`。
- 修复世界频道传输协议字段提取，确保贴图/表情文件通过 contentType 正确渲染。
- 修复 TypeScript 类型错误，优化空值安全判断。

## 0.1.9 - 2026-07-02

- 友链管理“详情”由行内浮动 popover 改为居中弹出的模态对话框，修复点击两次才出现、且仅闪现一下的问题。
- 更新同步逻辑，精简友链管理中的 URL 请求数量（“全部”标签页由 inbox/outbox 各遍历 5 种状态共 10 次请求，精简为每个收件箱各一次），减少加载时间。
- 更新为更详细的错误提示。
- 修复 pushedAt 被 updatedAt 覆盖的问题。
- 前端优化空值判断。
- 将 update_option 的 autoload 参数从 false 改为默认 yes。
- 进一步优化界面。

## 0.1.8 - 2026-06-24

- 修复友链星球中未接入站点误显示“删除 / 解除关系”按钮的问题：收紧前端删除入口判定，只有目标站点已接入插件且关系状态为 following / mutual 时，才允许显示“删除友链 / 解除关系”按钮。

## 0.1.7 - 2026-06-24

- 修复“全部”筛选中同一邀请同时出现在 inbox/outbox 时方向被覆盖，导致待审核邀请操作异常的问题。
- 待审核操作按钮拆分为独立的“通过”和“拒绝”渲染条件，避免只显示单个按钮。
- 微调操作列宽与按钮内边距，保证两个按钮同排显示且不被裁切。

## 0.1.6 - 2026-06-24

- 修复友链管理待审核邀请“通过 / 拒绝”按钮宽度与其他操作按钮不一致的问题。
- 操作列布局回到与 Typecho 端一致的自然按钮宽度，避免按钮被固定拉宽。

## 0.1.5 - 2026-06-24

- 修复友链管理待审核邀请在 WordPress 后台缓存旧资源时仍只显示“拒绝”的问题。
- 后台资源版本升级到 0.1.5，强制刷新 `wp-astrahub-admin.js/css` 缓存。
- 待审核邀请操作区显示“通过 / 拒绝”双按钮，审核弹窗文案同步为“通过友链邀请”。

## 0.1.4 - 2026-06-24

- 根治“我已关注”单向删除刷新后回弹：删除单向关注时只移除本站 -> 对方的关系边，不再走双方解除关系。
- “我已关注”删除不会再发送解除关系邮件；只有互相关注的“解除关系”才通知对方。
- 友链星球按关系状态分流：following 显示删除友链，mutual 显示解除关系。
- 保持待审核邀请的操作判定为 inbox + pending，避免“对方邀请我”时因为我已单向关注对方而缺少同意按钮。
- 修复友链管理待审核邀请操作区在窄列中只露出“拒绝”的问题，固定显示“同意 / 拒绝”双按钮。

## 0.1.3 - 2026-06-24

- 修复解除关系后只更新本地状态、未立即推送最新友链快照，导致刷新页面又从 Hub 旧关系边恢复为“我已关注”的问题。

## 0.1.2 - 2026-06-24

- 修复解除友链关系后，WordPress 本地友链可能因为 URL 尾斜杠、大小写或站点标记匹配差异没有删除，导致后台仍显示“我已关注”的问题。
- 修复“友链管理 > 对方邀请”在部分数据状态下只显示“拒绝”、不显示“通过”的问题。
- 修复“全部”筛选没有完整展示所有邀请状态的问题。
- 修复解除关系成功提示中的中文乱码。
