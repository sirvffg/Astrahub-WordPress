独立博客的连接层：把分散在各个博客上的友链关系串成一张可探索的网络，让你在线交换友链、沿关系发现同好圈子。

== Description ==

每个独立博客都有一个友链页，上面链着几个朋友，朋友又链着他们的朋友。但这些关系只存在于单个站点里，没有谁把它们串起来看。AstraHub（中文叫「星链」）做的就是这件事：把分散在不同博客上的友链关系、节点身份和公开动态，串成一张可探索的关系网络。

它不是又一个友链展示页。传统友链管理关心的是“我这个站挂了谁”，AstraHub 关心的是“成百上千个站连起来之后长什么样”——谁和谁互链了，哪些圈子天然抱团，哪些博主气质相近却还没发现彼此。

你可以把它理解成独立博客圈的「连接层」：它不取代 WordPress 或任何博客系统，而是在博客与博客之间搭一座桥，让好内容不再只靠搜索和运气被看见。接入的节点越多，这张网的发现效率就越高。

本插件是 AstraHub 在 WordPress 端的接入客户端，需要配合 AstraHub 主星（Hub）服务端使用。接入与数据同步均由站点管理员主动发起，接入前会展示明确的数据同步说明并需要勾选授权。

= 它能做什么 =

* **整合多生态的博主** —— 不论对方用的是 WordPress、Halo 还是其它博客系统，只要接入同一个星链网络，就处在同一张关系图里，跨平台彼此可见、相互发现。
* **在线自由交换友链** —— 告别“跑到对方网站找入口、填表单、等回信”的旧流程。看到心仪的博客，直接发起友链邀请，对方在自己后台一键审核通过，双向友链同步生效，整个换链过程不超过一分钟。
* **发现圈子与同好** —— 沿着“朋友的朋友”、标签和节点关系，快速触达更多与你气质相近的博客和创作者圈层，而不必一个个博客手动去翻。
* **关系图谱** —— 内置 3D 交互式星链图谱，把站点之间的友链与圈层关系可视化呈现，一眼看清谁在你的圈层里、谁处在网络枢纽位置。
* **圈层归属** —— 系统会根据站点信息与内容主题，把博客自动归入对应的星团（圈层），同圈层的博主彼此可见、相互发现。
* **迁移无忧** —— 换服务器或换域名后，可通过“重新登舱”恢复站点身份，原有的友链关系与圈层归属全部保留。
* **资讯聚合** —— 聚合阅读网络中其它站点的 RSS 资讯，关注圈友的最新动态。
* **前台挂件（可选）** —— 在前台展示一个吉祥物挂件与圈友入口，默认关闭，由管理员自行开启。

= 外部服务说明（External services） =

本插件会与第三方服务 **AstraHub 主星（Hub）** 通信，服务地址为 `https://astra.aobp.cn`。该服务是星链网络的中枢，负责聚合各站点的友链关系、撮合友链交换、构建关系图谱。

**何时发送数据：**

* 当管理员在“接入配置”中主动发起接入、更新信息、重新登舱、立即同步，或发起/审核友链邀请等操作时。
* 站点接入后，插件会按计划任务（每小时）自动把友链与关系数据上报主星，并把友链关系反向对账。
* 前台挂件开启且站点已接入时，前台页面会向插件自身的同源代理请求节点头像（由插件服务端代为拉取远端图片，避免跨域）。

**发送的数据范围：**

* 站点身份信息：站点名称、站点 URL、节点名称与头像、RSS 地址、站点语言。
* 友链数据：友链分组（名称），以及每一条友链的名称、链接地址、描述、图标与对应的 RSS 地址——这是构建跨站点关系图谱的基础。
* 联系邮箱：用于接收签发码、友链邀请与系统通知。
* 接入凭据（站点编号 / 密钥）：由主星签发，存储在本站点本地。

本插件只同步站点身份与友链关系数据，不会采集或上报你的文章（标题、正文、草稿等）以及任何后台私有内容；以上数据均来源于站点已对外发布、可被公众访问的公开信息。

== Installation ==

1. 在“插件 > 安装插件 > 上传插件”中上传本插件 zip，或解压到 `wp-content/plugins/wp-astrahub` 目录。
2. 启用插件后，左侧菜单出现“AstraHub 星链”。
3. 进入“接入配置”，填写站点信息，阅读并勾选数据同步说明后，通过邮箱签发码完成接入。
4. 接入成功后即可使用友链星球、友链交换、关系图、资讯等功能。

== Frequently Asked Questions ==

= 必须接入主星才能用吗？ =

是。本插件是 AstraHub 星链网络的接入客户端，核心功能依赖 AstraHub 主星服务端。未接入时只能看到接入配置页。

= 会自动向外发送我的数据吗？ =

只有在你主动接入后，才会同步上文“外部服务说明”中列出的公开内容与联系邮箱。接入前会展示完整说明并要求勾选授权，授权前不会发送任何数据。

= 接入需要付费吗？ =

接入与基础功能本身不向用户收取费用。具体以服务提供方的服务条款为准。

= 换了域名或服务器，原来的友链关系会丢吗？ =

不会。在“接入配置”中使用“重新登舱”恢复站点身份即可，原有的友链关系与圈层归属会保留。

= 如何关闭前台挂件？ =

前台挂件默认关闭。如已开启，在“接入配置 > 前台显示”中关闭“显示前台挂件”开关即可。
